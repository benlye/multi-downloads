MULTI-Module Firmware Downloads

This page provides an easy way to download the latest firmware file for your MULTI-Module, or MULTI-Module equipped radio. By default the latest firmware is shown, with older releases available using the drop-down Firmware Version selector

Once you have downloaded the firmware file for your module or radio, please refer to the MULTI-Module documentation for further instructions on updating the firmware.

Important: please click here to read about changes to the firmware builds and file filters starting from v1.3.1.77, including the radio firmware versions required to support the changes.

WARNING

Pre-release builds are intended for testing and validation of changes or new features in the MULTI-Module firmware.

It is extremely important that you read the release notes for the build you are testing, and that you fly with caution.

If you choose to test a pre-release build, please give feedback to the MULTI-Module developers via the forum.

WARNING

PPM firmware builds are examples which are intended for testing and verification only.

Due to the large number of available protocols and the limited number of protocol selection switch positions it is impossible to create PPM builds which will suit all users.

Users of PPM-mode modules are expected to configure and compile their own firmware using these instructions.

The protocols which are configured by default in the PPM builds can be found in the source code, in the firmware configuration file.

WARNING

Firmware builds for Atmega328p MULTI-Modules are intended for testing and verification only

Due to the small amount flash space available on the Atmega328p these builds only contain a subset of the available protocols.

Owners of Atmega328p-based modules are expected to configure and compile their own firmware using these instructions.

Release File Changes

Starting from MULTI-Module firmware version v1.3.1.77 the firmware files have changed and some options are no longer required.

  • No need to choose between OpenTX and erSkyTx/er9x builds because OpenTX and erSkyTX/er9x now use the same method to receive telemetry data from the MULTI-Module.
  • No need to choose between inverted and non-inverted telemetry because the radio and MULTI-Module will automatically negotiate the telemetry inversion setting

Firmware file names have been changed to reflect that these choices are no longer required.

The following minimum radio firmware versions are required in order to support these changes:

  • OpenTX - 2.3.9
  • erSkyTx - R223A4
  • er9x - v822U
